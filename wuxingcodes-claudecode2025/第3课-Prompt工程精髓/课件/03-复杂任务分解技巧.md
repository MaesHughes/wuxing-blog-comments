# 03-复杂任务分解技巧

## 📋 对应文案章节

对应《ClaudeCode纯干货教程第3课-Prompt工程精髓》第三部分：复杂任务分解技巧

## 🎯 技巧概述

本模块提供将复杂任务分解为可执行步骤的系统化方法，帮助高效管理和控制项目风险。

---

## 🏗️ 大项目分解策略

### 1. 分层分解法

**三层分解结构：**
```bash
# 第一层：功能模块分解
claude "请将【${PROJECT_NAME}】分解为主要功能模块：

**项目背景：** ${PROJECT_CONTEXT}
**项目规模：** ${PROJECT_SCALE}

请输出：
1. 核心功能模块
2. 支撑功能模块
3. 外围功能模块"

# 第二层：具体功能分解
claude "请将【${MODULE_NAME}】模块分解为具体功能：

**模块目标：** ${MODULE_OBJECTIVE}
**技术约束：** ${TECH_CONSTRAINTS}

请按优先级排序：
- 高优先级功能
- 中优先级功能
- 低优先级功能"

# 第三层：实现步骤分解
claude "请将【${FEATURE_NAME}】功能分解为开发步骤：

**验收标准：** ${ACCEPTANCE_CRITERIA}
**时间限制：** ${TIME_LIMIT}

请提供详细的开发步骤清单。"
```

### 2. 依赖关系分解

**任务依赖分析：**
```bash
claude "请分析【${PROJECT_NAME}】的任务依赖关系：

**任务清单：**
${TASK_LIST}

**分析要求：**
1. 识别前置依赖任务
2. 找出可以并行执行的任务
3. 确定关键路径
4. 制定执行顺序

**输出格式：**
- 依赖关系图
- 并行执行组
- 关键路径分析"
```

## ⚠️ 风险控制分解

### 1. 高风险任务识别

**风险评估Prompt：**
```bash
claude "请识别【${PROJECT_NAME}】中的高风险任务：

**评估维度：**
- 技术复杂度（1-5分）
- 依赖外部程度（1-5分）
- 时间紧迫性（1-5分）
- 资源充足度（1-5分）

**风险控制策略：**
1. 高风险任务的备选方案
2. 关键技术的前期验证
3. 并行开发的可能性
4. 监控和预警机制"
```

### 2. 分阶段实施策略

**渐进式分解：**
```bash
claude "请为【${PROJECT_NAME}】制定分阶段实施计划：

**阶段划分原则：**
- 每个阶段1-2周完成
- 每个阶段都有可交付成果
- 风险逐步降低
- 经验逐步积累

**阶段规划：**
## 阶段1：${PHASE1_NAME}
- 目标：${PHASE1_GOAL}
- 范围：${PHASE1_SCOPE}
- 验收：${PHASE1_ACCEPTANCE}

## 阶段2：${PHASE2_NAME}
- 目标：${PHASE2_GOAL}
- 范围：${PHASE2_SCOPE}
- 验收：${PHASE2_ACCEPTANCE}"
```

---

## 💡 实用技巧总结

### DO - 推荐做法
- ✅ 先做高层次的模块分解
- ✅ 识别任务间的依赖关系
- ✅ 制定风险控制策略
- ✅ 采用渐进式实施

### DON'T - 避免做法
- ❌ 直接进入细节实现
- ❌ 忽略任务依赖关系
- ❌ 低估复杂度和风险
- ❌ 缺少阶段性验收

通过这些分解技巧，可以将复杂的项目任务转化为清晰、可控、可执行的实施计划。
