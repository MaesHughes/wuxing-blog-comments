# 02-代码审查专用Prompt集合


## 📋 对应文案章节

对应《ClaudeCode纯干货教程第3课-Prompt工程精髓》第二部分：代码审查专用Prompt

## 🎯 审查类型概述

代码审查是确保代码质量和项目成功的重要环节，以下是针对不同审查场景的专用Prompt模板。


## ⚡ 性能审查Prompt

### 基础性能审查
```bash
claude "请对以下代码进行性能审查：

**性能指标：**
- [ ] 算法时间复杂度分析
- [ ] 空间复杂度评估
- [ ] 内存使用效率
- [ ] CPU利用率
- [ ] I/O操作优化
- [ ] 数据库查询性能
- [ ] 网络请求优化
- [] 缓存策略有效性

**代码：**
${PERFORMANCE_REVIEW_CODE}

**请提供：**
1. **性能瓶颈识别**：具体的性能瓶颈点和影响范围
2. **优化建议**：详细的性能优化方案
3. **预期性能提升**：量化的性能改善预期
4. **基准测试建议**：性能测试的具体方法和工具"
```

### 响应时间专项审查
```bash
claude "请重点审查以下代码的响应时间性能：

**响应时间要求：**
- API响应时间：${API_RESPONSE_TIME}ms
- 页面加载时间：${PAGE_LOAD_TIME}ms
- 用户交互响应：${INTERACTION_TIME}ms

**审查重点：**
- [ ] 数据库查询优化
- [ ] 缓存策略应用
- [ ] 异步操作处理
- [ ] 代码执行效率
- [ ] 网络延迟优化

**代码：**
${RESPONSE_TIME_CODE}

**请提供：**
1. **响应时间分析**
2. **延迟原因定位**
3. **具体优化措施"
4. **预期改善效果"
```


## 🔧 可维护性审查Prompt

### 代码质量审查
```bash
claude "请对以下代码进行代码质量审查：

**质量标准：**
- [ ] 命名规范和可读性
- [ ] 代码结构和组织
- [ ] 注释和文档完整性
- [ ] 错误处理机制
- [ ] 代码复杂度控制
- [ ] 单一职责原则
- [ ] 代码重复度
- [ ] 可测试性

**代码：**
${CODE_QUALITY_REVIEW_CODE}

**请提供：**
1. **质量问题清单**：按严重程度排序
2. **改进建议**：具体的改进方案
3. **重构建议**：重构的优先级和范围
4. **最佳实践应用**：推荐的编码实践"
```

### 技术债务识别
```bash
claude "请识别以下代码中的技术债务：

**技术债务类型：**
- [ ] 代码异味（Code Smells）
- [ ] 过时的技术栈
- [ ] 临时的解决方案
- [ ] 缺乏测试覆盖
- [ ] 不一致的编码风格
- [ ] 安全漏洞
- [ ] 性能问题
- [ ] 文档缺失

**代码：**
${TECHNICAL_DEBT_CODE}

**请提供：**
1. **技术债务清单**：按影响程度和修复成本排序
2. **修复优先级**：建议的修复优先级
3. **重构计划**：分阶段的重构计划
4. **预防措施**：避免技术债务积累的建议"
```
