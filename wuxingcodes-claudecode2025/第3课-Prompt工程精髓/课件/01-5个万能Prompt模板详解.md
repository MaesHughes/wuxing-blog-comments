# 01-5个万能Prompt模板详解

## 📋 对应文案章节

对应《ClaudeCode纯干货教程第3课-Prompt工程精髓》第一部分：5个万能Prompt模板

## 🎯 模板概述

这5个万能Prompt模板覆盖了日常开发中的主要场景，经过实际项目验证，可以直接使用或根据需要调整。

---

## 🏗️ 模板1：任务分解模板

**适用场景：**
- 任何复杂功能的开发和实现
- 大型项目的规划和管理
- 多人协作的任务分配

**使用效果：**
- 将复杂任务分解为可执行的步骤
- 明确各阶段的时间要求和验收标准
- 提高项目规划效率

### 完整模板
```bash
claude "请将【${TASK_DESCRIPTION}】分解为详细的实施步骤：

**任务目标：** ${MAIN_OBJECTIVE}
**技术栈：** ${TECH_STACK}
**约束条件：** ${CONSTRAINTS}

请按以下格式分解：
## 阶段1：${PHASE1_NAME}
- 步骤1.1：${DETAIL}
- 步骤1.2：${DETAIL}

## 阶段2：${PHASE2_NAME}
- 步骤2.1：${DETAIL}
- 步骤2.2：${DETAIL}

每个阶段完成后请等待我的确认再继续。"
```

## ⚡ 模板2：代码优化模板

**适用场景：**
- 代码性能优化
- 代码质量改进
- 代码重构和现代化
- 性能瓶颈解决

### 完整模板
```bash
claude "请优化以下代码，提升性能和可维护性：

**优化目标：** ${OPTIMIZATION_GOALS}
**性能要求：** ${PERFORMANCE_REQUIREMENTS}
**兼容性：** ${COMPATIBILITY_REQUIREMENTS}

**代码：**
${CODE_TO_OPTIMIZE}

**请提供：**
1. 优化后的完整代码
2. 具体优化说明
3. 性能提升预期
4. 潜在风险评估"
```

---

## 🔍 模板3：调试诊断模板

**适用场景：**
- Bug排查和问题解决
- 系统故障诊断
- 性能问题分析
- 代码错误修复

### 完整模板
```bash
claude "请帮我调试以下问题：

**问题现象：** ${PROBLEM_DESCRIPTION}
**错误信息：** ${ERROR_MESSAGE}
**复现步骤：** ${REPRODUCTION_STEPS}
**相关代码：** ${RELATED_CODE}

请提供：
1. 问题根本原因分析
2. 具体解决方案
3. 预防措施建议"
```

---

## 🔬 模板4：代码审查模板

**适用场景：**
- 代码质量检查和改进
- Pull Request审查
- 团队代码标准化
- 最佳实践推广

### 完整模板
```bash
claude "请对以下代码进行全面审查：

**审查重点：** ${REVIEW_FOCUS}
**业务背景：** ${BUSINESS_CONTEXT}

**审查维度：**
- [ ] 代码规范和风格
- [ ] 逻辑正确性
- [ ] 性能影响
- [ ] 安全性考虑
- [ ] 可维护性

**代码：**
${CODE_TO_REVIEW}

**请提供：**
1. 发现的问题清单
2. 修改建议
3. 改进后的代码示例"
```

## 📚 模板5：学习指导模板

**适用场景：**
- 新技术学习和理解
- 技能提升和培训
- 知识体系构建
- 学习路径规划

### 完整模板
```bash
claude "请帮助我学习【${LEARNING_TOPIC}】：

**当前水平：** ${CURRENT_LEVEL}
**学习目标：** ${LEARNING_OBJECTIVES}
**时间安排：** ${TIME_SCHEDULE}

**请提供：**
1. 学习路径规划
2. 重点概念解释
3. 实践练习建议
4. 进阶学习资源"
```

---

## 💡 使用技巧

1. **先理解再使用**：了解每个模板的应用场景
2. **灵活调整**：根据具体需求修改模板内容
3. **提供充分信息**：尽可能填写详细的变量值
4. **迭代优化**：根据使用效果不断调整模板

通过这5个万能模板，可以解决日常开发中的大部分场景需求。
