# 项目结构分析示例

哈哈，这些都是我接手老项目时的保命技能！看看我都是怎么快速搞懂一个项目的：

## 1. 项目结构分析命令

### 分析整个项目（刚接手项目必用）
```bash
# 我的天，这项目怎么这么乱，先看看结构
claude "帮我理理这个项目的结构，生成个目录树，要一目了然的那种"

# 这项目用的啥技术栈啊
claude "看看 package.json 和文件夹，告诉我这项目用的技术栈"
```

### 分析特定目录（深入某个模块）
```bash
# 看看组件都写了些啥
claude "分析下 src/components，告诉我都有哪些组件，它们之间有关系吗"

# API 都在哪写着呢
claude "查查 src/api 目录，把所有接口都列出来，要带说明的"
```

## 2. 依赖关系分析（避免踩坑必备）

### 查找文件依赖（改代码前必看）
```bash
# 看看主文件都依赖了啥
claude "分析 index.js，告诉我它都引入了哪些模块，我怕改错了"

# 最怕循环依赖了，赶紧检查
claude "帮我检查下有没有循环依赖，这个太坑了"
```

### 模块耦合度分析（重构前必看）
```bash
# 看看模块是不是耦合太紧了
claude "评估下这些模块的耦合度，如果要改要特别注意哪些"
```

## 3. 代码质量分析

### 代码组织评估
```bash
# 检查代码组织
claude "评估项目的代码组织是否合理，是否符合最佳实践"

# 识别重复代码
claude "查找项目中的重复代码，建议重构方案"
```
